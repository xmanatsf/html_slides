Attention-FFN Disaggregation (AFD) Solves a Structural GPU Underutilization Problem

Attention is stateful and KV-cache bound; FFN is stateless and SRAM-optimal — the system exploits both simultaneously

GPU Utilization: Attention vs FFN vs Batch Size

Attention → GPU (Stateful)

GPU HBM (288 GB) stores KV cache. Attention utilization barely improves with batch size — memory-bound. Disaggregation lets full HBM capacity be dedicated to KV cache, doubling token capacity per GPU.

FFN → LPU (Stateless)

LPU SRAM (500 MB) hosts FFN weights. FFN is deterministic with no KV state — inherently suited to LPU's static compute model. Each FFN expert is hundreds of MB; draft models need 256 GB DDR5 via FPGAs.

MoE tailwind: As frontier models become sparser, AFD value grows — more experts = fewer tokens per expert = greater benefit from disaggregation.

LPX Compute Tray Components

  • 16 LP30 LPUs (belly-to-belly on PCB — reduces X/Y trace lengths)
  • 2 Altera FPGAs ("Fabric Expansion Logic") — C2C-to-Ethernet bridge to Spectrum-X, C2C-to-PCIe to host CPU
  • 1 Intel Granite Rapids CPU + 1 BlueField-4 front-end module
  • 8× OSFP cages for cross-rack C2C; 2 QSFP-DD for GPU scale-out via Spectrum-X
  • PCB ASP ~$7k per main board (Victory Giant / WUS)

LPU Network Topology

  • Intra-tray: All-to-all mesh — each LPU connects to 15 others via 4×100G C2C (PCB traces)
  • Inter-node: 15×2×100G per LPU via copper cable backplane
  • Inter-rack: 4×100G per LPU to OSFP cages; daisy chain/ring for 4–8 racks
  • Scale-out (to GPUs): via Spectrum-X Ethernet through FPGAs

Nvidia's MBOM Design Strips Out OSFP Cages — Hyperscalers Are Saying "CP-Hell No"

Mid-board optical module replaces pluggable transceivers; hyperscalers demand OSFP cages for vendor flexibility

Nvidia's MBOM Vision

Midboard Optical Module (LGA socket) handles driver and TIA functions on-board. Two CX-9s share one MBOM. Provides 2 MPO ports at 2×800G each = 1.6T total per MBOM. Only 4× MPO ports per compute tray — no OSFP cages visible on GTC show floor Kyber rack.

Hyperscaler Pushback

MBOM blocks use of any pluggable transceiver or AEC cable. Hyperscalers are pushing back hard — they need OSFP cages to continue using their own pluggables and switch optical vendors independently. This is a significant architectural standoff with Nvidia's largest customers.

Nvidia is also exploring co-packaged copper (NPC) on the NVLink switch connector path as a loss-reduction measure. Supply chain is being told to prepare for fully co-packaged copper.

Vera ETL256 and CMX/STX: Nvidia Captures CPU and Storage Layers

Having locked compute and networking, Nvidia is now standardizing adjacent layers — expanding vertical integration systematically

V

Vera ETL256 — CPU Bottleneck Solved

256 Vera CPUs per single liquid-cooled rack

  • 256 CPUs per rack (32 compute trays × 8 Vera CPUs each)
  • Spectrum-X multiplane topology — 200 Gb/s lanes, single flat all-to-all Ethernet network
  • 32 front-facing OSFP cages for optical POD connectivity
  • Rationale: RL, code execution, and agentic workloads drive CPU demand independently of GPU scaling; CPUs are now a growing bottleneck
  • Same design philosophy as NVL rack — copper intra-rack, optics to POD
S

CMX / STX — Storage Layer Standardization

BF-4 DPU embeds in every compliant storage rack

  • CMX (ICMS rebranded) = "Tier G3.5" NVMe — sits between local SSD (G3) and shared storage (G4)
  • STX reference rack: 32 Vera CPUs, 64 CX-9 NICs, 64 SOCAMMs — endorsed by 15+ major storage vendors
  • STX mandates BlueField-4 DPUs in storage racks — expands NIC/DPU revenue attached to storage deployments
  • Strategic implication: BF-4/CMX/STX = Nvidia has now captured compute → network → storage layers

Supply Chain Realignment: FIT Emerges, Amphenol Faces Structural Displacement

GTC 2026 reveals non-obvious supply chain shifts — Nvidia captures connector IP, AlphaWave wins LP30 SerDes

FIT — Biggest Emerging Winner

Licensed Amphenol's VR NVL72 backplane + Paladin HD connectors. Appears to be leading the Kyber Voronoi connector competition. Positioned to win both LPX backplane and Kyber midplane volume.

AlphaWave (Qualcomm) — Surprise LP30 SerDes Win

Only SerDes IP provider qualified for Samsung SF4. Supplies 112G SerDes for LP30/LP35 C2C. Nvidia reverts to its own NVLink SerDes from LP40 (TSMC) — dependency ends after LP35.

Victory Giant / WUS — High-ASP PCB

LPX compute tray main board PCB: ~$7k ASP. Requires 52-layer M9 spec PCB. Belly-to-belly LPU layout demands premium HDI routing capability.

Amphenol — Licensing Revenue, Losing Design Seat

Earned licensing fees from FIT for Paladin/backplane tech. But Kyber "Voronoi" connector is Nvidia-proprietary — Amphenol is competing for a spec it no longer controls. Structural displacement from prior dominance.

Feynman Pushes Three Simultaneous Architectural Firsts — Highest Execution Risk in Nvidia's Roadmap

A16 + hybrid bonding/SoIC + CPO + custom HBM in a single platform; SK Hynix wins DRAM for LP40 3D stacking

A16

TSMC A16 Process Node

First Nvidia GPU platform on TSMC A16 — a next-generation process with backside power delivery (Super Power Rail). LP40 LPU also moves to TSMC N3P + CoWoS-R. Both represent first-of-kind node transitions co-timed with new system architecture.

High Exec Risk
3D

Hybrid Bonding / SoIC + Custom HBM

LP40 will use hybrid-bonded DRAM (SK Hynix as named supplier) to extend on-chip memory with near-SRAM performance characteristics. This replaces conventional DRAM packaging with die-to-die bonding — a technical first for the LPU platform.

SK Hynix Win
CPO

Co-Packaged Optics at Volume

NVL1152 (8× Kyber racks) is the first volume CPO deployment — for inter-rack scale-up. Jensen Huang's "all CPO" characterization likely refers to rack-to-rack only, not intra-rack. Two topology options: 2-layer CLOS fat tree or reconfigurable dragonfly with OCS.

Five Material Execution Risks That Could Alter the Investment Thesis

Risk register — ranked by probability and investment impact

🔴 AlphaWave LP30 SerDes Repeat (HIGH)

Same vendor that caused LPU Gen 2 failure is supplying Gen 3 SerDes. Report states issues "appear fixed" without yield data. Repeat failure would delay LPX production substantially.

🔴 Feynman Co-Execution Risk (HIGH, long-dated)

A16 qualification, hybrid bonding, CPO, and custom HBM all simultaneously in one platform. Any slippage pushes the first real CPO volume ramp further out, compressing TAM timelines.

🟡 Kyber Design Flux (MEDIUM-HIGH)

Voronoi connector spec, MBOM vs. OSFP, canister architecture — all explicitly in flux. "Rubin Ultra will be redone." Supply chain positioning for Kyber carries meaningful revision risk.

🟡 NVSwitch 7 Radix Uncertainty (MEDIUM-HIGH)

Supply chain evidence suggests NVSwitch 7 = same BW as NVSwitch 6. If true, NVL288 all-to-all topology is infeasible. Authors believe 2×BW/radix is more likely — unresolved. Impacts NVL288 cable demand by ~20,000+ DPs.

🟡 CPO Scope in NVL1152 (MEDIUM)

Jensen's "all CPO" vs. technical blog's "rack-to-rack only" is unresolved. Models pricing full intra-rack CPO would materially overestimate CPO content. Base case: inter-rack only.

Bull vs. Bear: The Inference Kingdom Expansion in Two Scenarios

The bull case requires execution on five simultaneous first-of-kind technologies by 2028

🟢 Bull Case

  • LP30 ramps without SerDes issues — adds true incremental inference capacity outside TSMC/HBM constraints
  • AFD becomes the inference standard for all interactive MoE deployments — every new frontier model drives LPX demand
  • Kyber volume ramp in 2027 with 144-GPU all-copper all-to-all scale-up; Voronoi connector locks in FIT/Amphenol volume
  • Vera ETL256 opens new TAM — CPU cluster demand grows independently from RL and agentic workloads
  • STX/CMX embedded in storage layer — BF-4 DPU volume attached to every storage deployment
  • Feynman delivers on schedule (2028) with first volume CPO + hybrid bonding — multi-year technology moat

🔴 Bear Case

  • AlphaWave SerDes fails again on LP30 — LPX production delayed; incremental capacity thesis doesn't materialize in 2026
  • Hyperscalers reject Kyber MBOM — design reverts to OSFP, pushing Kyber production later; supply chain locked on wrong components
  • NVSwitch 7 radix = same as NVSwitch 6 — NVL288 infeasible; world size scaling hits bottleneck at 144 GPUs until new switch generation
  • CPO economics don't work for NVL576 — similar to prior pluggable optics BOM blowout; NVL576 stays low volume indefinitely
  • Feynman slips 12–18 months — A16 yield issues or hybrid bonding delays; volume CPO ramp pushes to 2030+
  • AFD adoption limited — software complexity of ping-pong pipeline parallelism proves too burdensome for deployment at scale
